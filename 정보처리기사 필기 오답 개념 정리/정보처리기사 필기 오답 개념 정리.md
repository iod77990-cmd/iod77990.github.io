## 관계대수
원하는 정보와 그 정보를 어떻게 유도하는가를 기술하는 절차적인 방법

## 순수관계연산자
- select : 특정 조건을 만족하는 튜플 행 검색(수평연산)
- project : 특정 속성만 추출하여 새로운 릴레이션 생성(수직연산)
- join : 공통 속성을 기반으로 2개 릴레이션을 연결하여 하나의 릴레이션 생성
- division : 모든 속성을 가진 R의 튜플에서 S관련 속성을 제외한 나머지 

## 논리적 설계와 물리적 설계
- 논리적 설계 : 개념적 데이터 모델을 DBMS에 맞는 논리적 데이터 모델로 변환, DBMS의 특성을 고려하여 최적화 및 정제
- 물리적 설계 : 논리적 설계를 기반으로 실제 저장구조 설계(상세히)

## 병행제어 기법종류
- 최적 병행 수행
- 타임스템프
- 다중버전기법
-  Locking : locking 단위가 커지면 관리가 쉽고 병행성 수준이 낮아진다, locking 단위가 작으면 관리가 어렵고 병행성 수준이 높아진다.
   - 문제점 : 갱신분실, 비완료, 의존성, 모순성, 연쇄복귀 
   - 목적 : DB공유 최대화, DB일관성 최대화, 시스템 활용도 최대화, 응답시간 최소화

## 순차 다이어그램(Sequence Diagram)

## 데이터 흐름도(Data Flow Diagram)

## 럼바우 분석 기법
- 기능 : 여러 프로세스 간 자료 흐름을 표시(DFD)
- 동적 : 제어흐름등 상태를 시간 흐름에 따라 상태다이어그램으로 표시
- 객체 : 정보모델링이라고도 하고 시스템에서 요구하는 객체를 찾고 객체 간의 관계를 정의

## 객체지향기법에서의 관계성
- is member of : 연관성, 참조 및 이용관계
- is part of : 집단화, 객체 간의 구조적인 집약 관계
- is a : 일반화, 특수화, 클래스 간 개념적인 포함관계

## MOM(Message Oriented Middleware)
메시지 기반 비동기형 메시지를 전달하는 방식의 미들웨어
온라인 업무 보다 이기종 분산 데이터 시스템의 데이터 동기화를 위해 많이 사용됨

## 에자일 방법론
- 스크럼
고객의 요구사항을 초점을 맞추고 짧은 개발 주기를 갖고 점진적이며 경험적으로 시스템을 지속해서 개발/전달, 2 ~ 4주간 주기의 스프린트의 반복으로 backlog, planning, scrum, review, retro 순서로 구성
계획하는 단계에 회의를 진행하고 소멸차트를 이용하여 작업의 진행도를 시각화 하고 남은 작업량을 시각화

-  XP(eXtreme Programming)
개발주기를 3 단계로 나눠 요구 사항을 반영하고 개발
  - 탐색
  - 몰입
  - 조정

- 크리스탈
팀과 프로그램 특성에 맞춰 다양한 접근 형식 적용

- 적응형
복잡하고 변화하는 요구사항 대비
- 기능중심

- 익스트림 모델링
모델링과 설계를 단순화하고 반복적, 점진적으로 진행

- 칸반
작업의 흐름을 시각화하고 관리

## 인터페이스 
시스템의 구조와 서브시스템들 사이 관계를 표현, 순서적 연산에 의해 소프트웨어를 실행하는 절차

## 클래스, 캡슐화, 인터페이스
- 클래스 : 속성과 관련된 연산을 묶어서 하나로 취급하는 것을 의미, 객체 생성 가능(Class), extends로 단일상속 
- 캡슐화 : 데이터 보호를 위해 접근지정자로 지정해 보호하는 것을 의미
- 인터페이스 : 메서드 규칙만 정의한 시그니쳐, 기능 규칙 정의, 객체 생성 불가능(interface), implements로 다중상속

## 알고리즘 설계 기법
- Divide-Conquer(분할정복)
- Dynamic(동적계획)
- Greedy(탐욕)
- Backtracking (퇴각계획)
- Branch&Bound (분기한정)
- Approximation (근사)

## 유스케이스(UseCase)에서 엑터(Actor)
엑터는 서비스를 이용하는 외부객체로 시스템이 특정한 사례를 실행하도록 요구할 수 있는 존재

## 유스케이스 다이어그램
시스템과 사용자의 상호작용 표현
- 엑티비티 : 일련의 활동들을 도식화
- 시퀀스 : 특정 이벤트가 어떤 순서로 어떤 객체와 어떻게 상호작용하는지 표현

## N-S차트(Nassi-Schneiderman)
선택, 조건, 반복조건을 구조적 흐름으로 표현

## HIPO(Hierarchy Input Process Output)
- 프로그램의 기능을 계층구조로 도식화, 개발순서를 논리적으로 전개
- 하향식중심, 절차보다 기능 중심, 표준화된 문서 작성기법을 사용해 의사전달 착오가 적다
- 종류 : 가시적도표, 총제적도표, 세부적도표
- 프로그램 구조와 데이터 구조, 데이터 구조간 관계를 표현할 수 있다
- 기본시스템은 입력, 출력, 처리로 구성

## 워크스루(Workthrough)
단순한 테스트케이스를 이용하여 프로젝트를 수작업으로 수행해 보는 것

## 케이스

## 테스트 케이스(TestCase)
구현된 소프트웨어가 사용자의 요구사항을 정확하게 준수했는지 확인하기 위해 설계된 입력, 실행 조건, 기대결과 등으로 구성된 테스트 항목에 대한 명세서

## 화이트박스 테스트에서 기본경로(Basic path)
제어흐름그래프를 분석하여 선형독립실행경로를 찾고 맥케이브 순환복잡도를 사용하여 테스트 사례 생성

## 인스펙션(Inspection)
소프트웨어 요구, 설계, 원시코드 등의 작성자 외의 다른 전문가 또는 팀이 검사하여 오류를 찾아내는 공식적인 검토 방법

## 선형검색(Linear Scanning)
- 원하는 레코드를 찾을 때까지 처음부터 끝까지 하나씩 비교하면서 검색
- 평균 검색 횟수 (n + 1)/2
- 순차검색 이라 한다
- 정렬되지 않는 검색에 가장 유용, 평균 검색 시간이 오래 걸림

## IP프로토콜에서 사용되는 필드 특징

- Total Packet Length : 전체 패킷 길이를 byte 단위로 표시, 헤더와 데이터(페이로드)를 더한 것, 최대크기는 65,535byte (2^16 - 1)
- Time to Live : 패킷을 전달할 수 있는 횟수의 제한을 나타낸 것

  - 요소를 수정, 삭제, 추가 를 할 수 없음

  - "{ }"

## 스레드(Thread)
프로세스 내에서 작업단위로 시스템의 여러 자원을 할당받아 실행하는 프로그램 단위

## Authentication
시스템에 사용자의 신원을 증명하는 과정

## 직접파일(Direct File)
- 직접접근 기억장치의 물리적 주소를 통해 직접 레코드에 접근하는 파일구조
- 해싱 등 사상함수로 레코드 키에 의한 주소계산을 통해 레코드에 접근
- 키에 일정 함수를 적용하여 상대 레코드 주소를 얻고 그 주소를 레코드에 저장

## Recovery(복구)
- Deferred Modification : 변경된 데이터를 실제 디스크에 반영을 지연
- Immediate Update : 데이터 병경사항을 즉시 디스크에 반영
- Shadow Paging : 복구를 위해 일부 페이지를 원래 페이지와 별도의 그림자 페이지 유지
- CheckPoint  : 특정시점에 상태기록

## 교착상태 발생 조건
- 상호배제(Mutual Exclusion)  : 한번에 하나만 공유 자원 사용
- 점유와 대기 : 자원을 사용중인 프로세스가 다른 자원을 요구
- 비선점 : 할당된 자원은 강제로 뺏을 수 없음
- 환형대기 : 할당된 프로세스가 앞이나 뒤로 프로세스 자원을 요구

## 디자인 패턴
생성패턴 ( 추상 객체 인스턴스화 )
- 추상팩토리 (Abstract Factory) : 객체 생성 과정을 추상화한 인터페이스 제공, 관련있는 여러 객체를 구체적인 클래스 하나하나에 의존하지 않고 추상화를 집합적으로 묶음(Factory 1개 당 관련있는 객체 생성 지원)
- 팩토리메소드 (Factory Method) : 객체 생성 과정을 추상화한 인터페이스 제공, 구체적인 객체 생성과정을 하위 또는 구체적인 클래스로 옮김(1개의 Factory 당 1종류의 객체 생성 지원)
- 빌더 (Bulider) : 복잡한 객체의 생성과 표현을 분리해서 확장 가능한 인스턴스 생성
- 프로토타입 (Prototype) : 복제 가능한 객체
- 싱글톤 (Singleton) : 메모리 절약을 위해 같은 인스턴스 생성 시 기본의 인스턴스를 가져와 활용

구조패턴 ( 객체 결합 )
- Bridge : 기능 클래스와 구현클래스를 연결하고 구현부에서 추상계층을 분리하여 독자적인 확장 및 변경을 가능한 패턴
- 어댑터 (Adapter) : 호환성이 없는 인터페이스를 함께 동작할 수 있도록 변환, Wrapper 패턴, implements 로 인터페이스 상속(인터페이스끼리 다중상속), extends 로 클래스 상속(단일상속)
- 컴포지트 (Composite) : 복합 객체와 단일 객체 를 동일한 컴포넌트로 취급하고 동일한 인터페이스 사용
- 데코레이터 (Decorator) : 대상 객체에 대한 기능 확장이나 변경이 필요할 때 객체간 결합을 통해 사용
- 파사드 (Facade) : 복잡한 클래스 라이브러리를 간단한 인터페이스로 재정리
- 플라이웨이트 (FlyWeight) : 재사용 가능한 객체 인스턴스를 공유시켜 메모리 사용량 최소화, 경량패턴
- 프록시 (Proxy) : 원본 객체를 수정할 수 없을 때 대리하여 대신 처리하게 함으로 논리의 흐름을 제어하는 패턴

행위패턴 ( 객체 간 커뮤니케이션 )
- 옵저버 (Observer) : 대상자의 변화가 있을 때 서로다른 관찰자들에게 통지하고 조치를 취하는 패턴, 1:N패턴 의존성
- 전략 (Strategy) : 실행 중에 알고리즘 전략을 선택하여 객체 동작을 실시간으로 바꾸는 패턴
- 커멘드 (Command) : 어떤 동작을 요청하는 객체와 그 요청을 처리하는 객체를 분리하는 패턴
- 상태 (State) : 조건문의 상태가 아닌 특정한 상태를 객체화하는 패턴
- 책임연쇄 (Chain of Responsibility) : 여러 처리 객체를 나누고 사슬처럼 연결해 집합 안에서 연쇄적으로 처리하는 패턴
- 방문자 (Visitor) : 객체의 구조와 그 구조에서 수행되는 작업을 분리해서 확장성과 유연성을 높이는 패턴
- 인터프리터 (Interpreter) : 문법적인 규칙을 클래스로 정의하여 표현식을 해석하는 방식의 패턴
- 메멘토 (Memento) : 캡슐화를 위배하지 않고 객체 내부 상태를 실체화시켜 저장된 상태로 되돌아오는 패턴
- 중재자 (Mediator) : 객체간 직접 호출이 아닌 상호작용을 캡슐화하는 패턴
- 탬플릿 메서드 (Template Method) : 여러 클래스에 공통으로 사용하는 메서드를 탬플릿화해서 상위 클래스에 정의하고 하위 클래스마다 세부 동작을 다르게 구현하는 패턴
- 반복자 (Iterator) : 순서가 정해지지 않은 데이터 집합에 대해 순차적인 접근(순회) 하는 패턴

## 정렬된 N개를 처리하는데 시간복잡도
- O(1) : 해시함수
- O(log n) : 이진탐색
- O(n) : 순차탐색
- O(n^2) : 버블정렬, 삽입정렬, 선택정렬, 퀵정렬(최악)
- O(2^n) : 지수시간
- O(n log n) : 힙정렬, 퀵정렬(평균)
- O(nlog2n) : 퀵정렬, 합병정렬(평균, 최선, 최악 모두 동일)

## N개 정렬 방법
- 완전이진트리 : 힙정렬
- 분할정복 + 병합 : 합병정렬
- 인접 비교 반복 : 버블정렬
- 하나씩 삽입 : 삽입정렬
- 피벗기준 분할 : 퀵정렬
- 최솟값 선택 : 선택정렬
- 자릿수 : 기수정렬
- 카운팅 배열 : 계수정렬

## 정렬 
- 선택정렬
배열 전체에서 최솟값을 찾고 그 값을 현재위치와 바꾸고 pass 진행(앞에서 정렬 영역 확정)
- 버블정렬
인접한 두값을 비교해서 앞이 더 크면 서로 위치를 바꿈 마지막까지 비교후 pass 진행(뒤에서 정렬 영역 확정)

## 관계형 데이터 모델 릴레이션 특징

- 튜플과 속성의 순서는 의미가 없다

## 방화벽 구조
1. 패킷필터링 : 기본적인 방화벽 기능(IP주소, 포트번호, 프로토콜, 방향 검사)
2. 스크리닝라우터 : 패킷필터링 기능을 수행하는 라우터(인터넷-> 스크리닝 라우터-> 내부망 구조)
3. 스크린트서브넷 : 외부와 내부 사이에 완충지대를 만들어 보안 제공(외부망-> 스크리닝 라우터-> DMZ(베스천 호스트(웹서버 + DNS서버))-> 스크리닝 라우터-> 내부망)
4. 베스천 호스트 : 외부와 내부 네트워크 사이에 위치하는 강화된 보안 서버(공격에 노출됨)
  - Dual-Homed Host : 베스천 호스트가 방화벽 역할 까지 수행(인터넷-> 베스천 호스트-> 내부망), Gateway역할 수행
  - Screened Host : 패킷필터링 + 베스천 호스트(인터넷-> 스크리닝라우터-> 베스천 호스트(외부/내부NIC)-> 내부망), Gateway역할 수행

## 암호화 알고리즘
- 대칭키(Symmetric Key)
  - 같은 키로 암호화, 복호화, 키의 개수는 n(n-1)/2 개
  - AES, DES, SEED, IDEA, RC4
- 공개키(비대칭 키), 키의 개수는 2 x n 개
  - 키가 2개(공개/비밀)
  - RSA, ECC, DGA, Elgamel, Diffie-Hellman, ...
- 해시함수
  - 일방향 해시함수로 암호화, 복호화 불가능
  - SHA-256, ..., MD5...

## IPsec(IP Security)
네트워크계층에서 IP는 패킷을 암호화 인증

- "[AH]" Authentication Header : 데이터 변조 방지, 송신자 인증

- "[ESP]" Encapsulating Securuty Payload : 데이터 암호화, 인증/무결성 포함가능

- "[Mod]" Transport/Tunnel : 데이터만 보호하고 IP헤더는 그대로/패킷전체를 감싸서 새로운 IP헤더에 붙임(기존IP를 숨김, VPN에서 사용됨)

- "[IKE]" 키교환 : 암호화에 사용할 키를 안전하게 교환하는 프로토콜

## 문맥교환(Context Switching)
다중프로그래밍 시스템에서 CPU에 할당되는 프로세스를 변경하기 위해 운영체제가 현재 CPU에 상태 정보를 저장하고 앞으로 실행될 상태정보를 설정한 다음 CPU에 할당하여 실행되도록 하는 작업

## 정보보안의 3요소
- 기밀성(Confidentiality)
인가받지 않은 사용자는 정보를 알수 없도록 하는 특징
- 무결성(Integrity)
인가받은 사용자만 시스템 자원에 접근해서 수정할 수 있는 특징
- 가용성(Availability)
인가받은 사용자만 어디서든지 시스템 자원에 접근 가능한 특징

## 이분검색 방법
- 첫번째 원소 Low, 마지막 원소High, 중간원소를 (Low+High)/2, 찾고자 하는 값 key
- key > 중간값 : Low를 중간값 + 1로 두고 반복
- key < 중간값 : Low를 중간값 - 1로 두고 반복
- key = 중간값 : 검색 완료
