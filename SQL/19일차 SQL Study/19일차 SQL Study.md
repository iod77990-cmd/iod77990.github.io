### SQL 집계함수 사용 위치

**⭕ 사용 가능한 곳**

- **select**절
그룹화된 데이터의 결과값을 출력할 때 

- **having**절
집계 결과에 조건을 걸고 싶을 때, where절과는 차이점이 그룹화 된 이후의 결과를 필터링 하는 점

- **order by**절
집계된 숫자를 기준으로 순서를 정렬할 때 사용

**❌사용 불가능한 곳**

- **where**절
데이터를 그룹으로 묶기 전에 개별 행을 검사하는 단계로 계산되지 않은 집계 값을 조건으로 쓸 수 없음

- **group by**절
그룹을 나누는 기준 칼럼이 오는 자리로 계산 결과가 오는 자리가 아님

- **from**절
어떤 테이블에서 가져올지에 대한 자리

### 테이블 분리를 통해 정보 간 의존도 줄이기
1. 데이터 중복을 피하고 효율적인 데이터 관리를 위해 
테이블 1개를 성격이 비슷한 정보끼리 떼어 새로운 테이블로 만들어 관리

2. 테이블 간의 관계는 참조키만 알면 되게 하여
 정보간 의존도를 낮추고 서로 다른 테이블 간 정보는 참조키를 가지고 확인하면 된다.

### 조인(Join)
2개 이상의 테이블을 공통된 칼럼을 기준으로 결합해 하나의 결과로 만드는 연산

1. left join
  - Α ⊃ Β 

  - B가 속하지 않는 A의 부분집합

2. outer join
  - A ⋃ B (합집합)

  - A ⋂ B은 포함하지 않는 부분집합

3. right join
  - A ⊂ B 

  - A가 속하지 않는 B의 부분집합

4. inner join
  - A ⋂ B (교집합)

5. self join
하나의 테이블을 자기 자신과 조인


6. natural join
이름이 같은 칼럼을 자동으로 기준으로 한 조인


7. theta join
2개의 테이블을 임의의 조건(비교연산자)으로 결합하는 조인


8. Equi join
세타 조인 중에서 = (같다) 조건만 사용하는 경우


9. semi join
한쪽 테이블의 행 중에서 다른 테이블에 존재한는 것만 필터링


10. anti join
다른 테이블에 존재하지 않는 데이터만 조회


---

- 명시적 조인(Explicit)
join명령어를 통해 명시적으로 table을 연결하는 방식
```sql
select * from A join B on A.id = B.id;
```
이점) 
1. 가독성 이 좋음
2. join 유형을 지정할 수 있다
3. join 조건이 명확하다

- 암시적 조인(Implicit)
where 명령어를 통해 암시적으로 table을 연결하는 방식
```sql
select * from A, B where A.id = B.id;
```
이점) 
1. 간결하다
2. inner join만 사용하게 된다
3. 가독성이 떨어진다.
