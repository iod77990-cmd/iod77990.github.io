## 관계형 DB 설계

DB는 구조화된 정보의 집합체로 DB를 효율적으로 관리하기 위한 설계 방법

### 모델링
- 현실의 모습을 DB로 추상화 하는 행위

  - 순서
    1. 요구사항 파악
    2. 개념적 설계
    3. 논리적 설계
    4. 물리적 설계

- PK(Primary Key, 기본키)

테이블의 각 레코드를 고유하게 식별
null값을 가질수 없고 테이블 당 하나만 존재 가능

- FK(Foeign Key, 외래키)

다른 테이블의 기본 키를 참조
테이블 간의 관계를 설정

### DB 테이블 간의 관계

종류)
- 1:1 관계
하나의 레코드가 다른 테이블의 레코드 한개와 연결된 경우

- 1:N 관계
하나의 레코드가 서로 다른 여러 개의 레코드와 연결된 경우

- N:M 관계
여러 개의 레코드가 다른 테이블의 여러 개의 레코드와 관계가 있는 경우

- 자기참조 관계
테이블 내에서의 관계

### DB 정규화 과정
데이터 중복을 최소화하고 데이터 무결성을 향상시키는 과정, 1NF, 2NF, 3NF가 주로 사용됨
